# 全局规则 — 扩展 Markdown 语法参考

## 朴素写作规则

非用户主动要求，正文及表格中的内容**禁止使用任何行内格式标记**，包括但不限于：

| 禁止   | 示例       |
| ------ | ---------- |
| 加粗   | `**文本**` |
| 斜体   | `*文本*`   |
| 删除线 | `~~文本~~` |
| 高亮   | `==文本==` |

### ❌ 禁止

```
该方法取得了**显著的**性能提升，*远超过*现有方法。
| ResNet | **76.1%** |
```

### ✅ 正确

```
该方法取得了显著的性能提升，远超过现有方法。
| ResNet | 76.1% |
```

> 注意：公式（`$...$` / `$$...$$`）也禁止外层包裹 `**...**`，公式字体由 Word 模板样式统一控制，加粗标记无效且可能干扰解析。

---

## 换行与段落分隔

### ❌ 禁止连续行书写

Markdown 中连续的多行（行间无空行）会被解析为 Word 软换行（`↓`），而非独立段落：

```
这是第一项
这是第二项
这是第三项
```

输出为：`这是第一项↓这是第二项↓这是第三项`

### ✅ 每项之间必须空一行

```
这是第一项。

这是第二项。

这是第三项。
```

## 行内格式（仅限语法参考）

| 格式 | 写法       |
| ---- | ---------- |
| 加粗 | `**加粗**` |
| 斜体 | `*斜体*`   |

> 根据「朴素写作规则」，实际写作中不主动使用加粗/斜体。

## 标签前缀约定

| 前缀   | 用途 | 声明方式                 |
| ------ | ---- | ------------------------ |
| `fig:` | 图片 | `@figure[fig:xxx]{描述}` |
| `tbl:` | 表格 | `@table[tbl:xxx]{表名}`  |
| `eq:`  | 公式 | `@formula[eq:xxx]`       |

---

## 四大核心规则

1. 每个图/表必须有 `@figure[标签]{描述}` / `@table[标签]{表名}` 紧接在前
2. 每个块公式必须有 `@formula[标签]` 紧接在 `$$...$$` 之前
3. `@ref{标签}` 的标签必须与声明的标签完全一致
4. `@cite{key}` 的 key 必须存在于 `.bib` 文件中

---

## 图片

### ✅ 正确写法

```
@figure[fig:架构图]{本文提出的端到端模型架构}
![04-01.png](assets/04-01.png)
```

### ❌ 禁止

- 缺少 `@figure[fig:xxx]{描述}` 声明
- 标签前缀不是 `fig:`

---

## 表格

### ✅ 正确写法

```
@table[tbl:对比]{不同方法性能对比}
| 方法 | Top-1 准确率 | 参数量 |
| --- | --- | --- |
| ResNet-50 | 76.1% | 25.6M |
| ViT-B/16 | 77.9% | 86.6M |
```

- 表格生成三线表格式

### ❌ 禁止

- 缺少 `@table[tbl:xxx]{表名}` 声明
- 标签前缀不是 `tbl:`
- 表格单元格内出现 `$$...$$` 块公式
- 表格单元格内使用加粗 `**...**`（违反朴素写作规则）

---

## 公式

### 行内公式

嵌入段落或表格中的公式，使用 `$...$` 包裹。

#### ✅ 正确写法

```
损失函数由 $L = L_{\text{cls}} + \lambda L_{\text{reg}}$ 定义。
混凝土强度为 $f_c = 14.3\text{N/mm}^2$。
```

#### ❌ 禁止

- 单独成行的公式使用 `$...$` — 必须用块公式 `$$...$$`
- 表格中出现 `$$...$$` — 表格内只能用 `$...$`
- 单位裸露在 `$...$` 外（如 `$f_c=14.3$N/mm²`，应写为 `$f_c=14.3\text{N/mm}^2$`）
- 百分号前加转义符 — 直接写 `%`，不要 `\%`

### 块公式

独立占行的公式，使用 `@formula[eq:xxx]` + `$$...$$`。

#### ✅ 正确写法

```
@formula[eq:accuracy]
$$
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
$$
```

#### ❌ 禁止

- 省略 `@formula[eq:xxx]` 声明
- 标签前缀不是 `eq:`
- 使用 Mathtype 无法识别的 LaTeX 命令：

| 禁止命令                            | 替代方案                               |
| ----------------------------------- | -------------------------------------- |
| `\dfrac{}{}`                        | `\frac{}{}`                            |
| `\cfrac{}{}`                        | `\frac{}{}`                            |
| `\begin{align}...\end{align}`       | 拆分为多个独立块公式                   |
| `\begin{equation}...\end{equation}` | `@formula` + `$$...$$`                 |
| `\displaystyle`                     | 块公式默认已是 display style，无需声明 |

## 交叉引用

### ✅ 正确写法

```
如 @ref{fig:架构图} 所示，结果见 @ref{tbl:对比}，目标函数见 @ref{eq:损失函数}。
```

### ❌ 禁止

- `@ref{xxx}` 中的标签与声明的标签不一致

---

## 参考文献

### ✅ 文中引用

```
相关研究 @cite{resnet2016,transformer2017} 表明……
```

- 多个引用用逗号分隔
- 生成上标 `[1,2]` 格式

### ✅ 参考文献列表

```
@bibliography{refs.bib}
```

- 放在文档末尾
- 自动生成 GB/T 7714 格式参考文献

### ❌ 禁止

- `@cite{key}` 的 key 不存在于 `.bib` 文件中
